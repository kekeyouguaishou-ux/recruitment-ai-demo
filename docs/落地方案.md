# 基于AI技术的招聘数据提效落地方案

## 1. 目标与范围

本方案针对HR在企业微信、企业微信群和腾讯在线文档之间重复记录招聘数据的问题，通过轻量AI应用完成简历自动结构化、候选人建档、招聘阶段更新、面试反馈同步和招聘数据可视化。

第一阶段不更换现有招聘工具，不让AI直接决定候选人是否录用，以最小成本验证节省录入时间、提高数据完整性和缩短更新时延的效果。

## 2. 用户流程

1. HR上传脱敏简历。
2. 系统解析文档并调用大模型，输出固定Schema。
3. HR确认低置信度或缺失字段。
4. 系统按手机号或邮箱去重，新增或更新候选人记录。
5. 内部筛选人员通过标准化操作更新招聘阶段。
6. 面试官提交评分、结论和反馈。
7. 当前数据表、事件流水和招聘看板实时更新。
8. 系统根据已计算指标生成招聘日报和待办建议。

## 3. AI应用点

### 简历结构化抽取

大模型将非结构化简历转换为`CandidateExtraction` Schema，包括姓名、联系方式、岗位、学历、学校、专业、工作年限、技能、渠道、置信度和缺失字段。模型温度设为0，并要求只返回JSON。

### 招聘日报生成

程序先确定性计算候选人数、阶段分布、面试比例、通过率、平均停留时长和超时待办，再让大模型负责自然语言总结。模型不直接读取个人敏感信息，也不作出录用决定。

## 4. 更新与实时同步

系统采用“当前快照 + 事件流水”双模型：

- 当前快照保存候选人的最新阶段和业务字段。
- 每次操作追加事件，记录原阶段、新阶段、业务发生时间、操作人、同步目标、同步状态和同步完成时间。
- `version`字段用于识别并发更新；`event_id`可作为后续幂等键。
- `next_followup_at`用于产生超时提醒。

公开Demo中，每次操作会立即写入SQLite并刷新页面，模拟腾讯文档API写入后的近实时效果。获得企业账号后，保持相同适配器接口，将写入实现替换为腾讯文档企业版API。

## 5. 数据可视化与决策支持

看板展示候选人总数、待筛选人数、面试推进人数、Offer数、超时待办、招聘阶段分布、岗位分布、面试比例、面试通过率和平均阶段停留时长。

数据用于发现流程瓶颈、安排HR处理顺序和评估招聘渠道，不用于自动评价候选人的个人能力。

## 6. 效果评估

建议在真实试点中记录以下基线与上线后指标：

- 单份简历平均录入时长
- 候选人核心字段完整率
- 招聘状态平均更新时延
- 重复候选人记录比例
- 超时未跟进数量
- 面试反馈平均回收时长
- HR人工修正率
- 单次简历抽取的模型成本

## 7. 风险控制

- 公开演示只使用虚构或脱敏数据。
- 联系方式在页面展示时脱敏。
- API Key通过环境变量或部署Secrets管理。
- 低置信度和字段缺失必须人工确认。
- 所有状态变化保留事件记录，支持审计。
- AI不直接决定淘汰或录用。

## 8. 迭代路线

第一阶段完成当前Demo闭环；第二阶段在获得测试权限后接入腾讯文档API和企业微信回调；第三阶段补充接口失败重试、权限控制、多人并发和真实业务指标评估。
